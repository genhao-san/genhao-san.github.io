?c=eval(next(reset(get_defined_vars())));&pay=system("tac flag.php"); //get_defined_vars()用于以数组的形式返回所有已定义的变量值（包括URL屁股后面接的pay），这里源码只定义了一个变量即c，加上你引入的a就两个变量值了。reset用于将指向返回变量数组的指针指向第一个变量即c，next向前移动一位指针即a，eval执行返回的值就是咱们定义的恶意代码。

分析代码，system($c." >/dev/null 2>&1")前面>/dev/null会将标准输出重定向到/dev/null这是系统中一个空设备，也就是“丢弃标准输出”。2>&1里面2代表标准错误输出1表示标准输出，&表示引用。也就是把标准错误输出也重定向到“标准输出当前指向的位置”即也丢弃。说这么多也就是屏幕上不会输出内容。但是我们可以用;来进行分割，使语句有执行顺序。

这里介绍一下转义字符和控制字符。

转义 % 自身的编码：若要在 URL 中传递字面量 % ，需用 % （或 %25 ， % 的 URL 编码是 %25 ）。此时 % 会被解析为 % 。

控制字符”的编码：像 %09 （代表 Tab 键）、 %0A （换行）、 %0D （回车）等，属于“对不可见控制字符的 URL 编码”，这类编码的作用是将特殊字符转换为 URL 安全的格式，和“转义 % 自身”完全无关。

没事我们海油绝招${IFS}

  • ${IFS} 的含义：IFS 全称为 “Internal Field Separator”（内部字段分隔符），默认值包含空格、制表符和换行符。在命令行里，${IFS} 可以用来替代空格。
  • 原因：正则表达式过滤了空格字符 ，但没有对 ${IFS} 进行过滤。
  • 所以，当把命令写成 ls${IFS}-l 时，在正则表达式的检查中，因为不存在被过滤的空格字符，该命令就能通过检查。
